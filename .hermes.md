# GLM-5.2 Integration Controls
- Limit file system reads to 1 file at a time sequentially. Do not batch read concurrently.
- When generating complex code files, pause for 2 seconds between sequential file writes.
- Rely strictly on local context memory over continuous workspace index searches.
- NEVER rewrite an entire code file if making small modifications.
- Use unified diff format (with clear context lines) or targeted function replacements.
- If an output is expected to be long, explicitly tell the user you will output it in parts, and pause for instructions.
- When executing terminal scripts, only inspect the last 50 lines of standard output.
- Do not dump massive raw test logs into the dialogue memory.
